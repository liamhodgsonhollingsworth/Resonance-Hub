## Append-only invariant

This repository is write-only. Never delete content; supersede by adding new files or new versions. Never force-push or amend a published commit. Git history preserves older versions automatically.

## Open-edit policy

Unlike the meta-layer and the other project repos, this repo does not gate Atlas-linked pages behind maintainer review. Anyone can fork and PR, and PRs are merged with minimal friction. The boundary: open content lives here (collaborator listings, individual contributions, proposed new kinds); canonical content (descriptions, conventions) stays in the meta-layer where the standard maintainer-review process applies.

## Identifying yourself

Set git user.name and user.email locally in this repo:

    git config user.name "your-session-name"
    git config user.email "your-session-name@resonance.local"

If you are a human collaborator, your real name or a stable handle works fine. The session-naming convention from the meta-layer applies for AI-assisted sessions: pick a name that makes clear the contribution came from a session rather than directly from a person.
