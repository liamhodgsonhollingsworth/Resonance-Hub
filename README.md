# Resonance Hub

The open collaborator entry point for the [Resonance meta-layer](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront) network. This repository is public and anyone can edit it without needing prior permission. Its job is to be the low-friction social surface where collaborators arrive, see what is possible, and add their own contributions.

The canonical content describing what the system can be, the contribution mechanics, and the project's conventions lives in the meta-layer repository. This hub primarily links back to that canonical content; new content (collaborator listings, individual contributions, new kinds-of-project proposed by collaborators) lives natively here, where it can be edited freely without affecting the canonical sources.

## Open-edit policy

This repository is fully open to public editing. The meta-layer's edit-permissions convention (Atlas-linked pages are maintainer-edited only) **does not apply here**; this repo overrides that convention because its role is the open collaborator surface.

Anyone can fork this repository and open a pull request adding new content. The repository does not use CODEOWNERS or branch-protection rules that require maintainer review. PRs are accepted with minimal friction; the maintainer reviews only for clear conflicts with the project's purpose or for content that should clearly live elsewhere (e.g., implementation details that belong in a project repo, canonical descriptions that belong in the meta-layer).

The rule of thumb for what belongs here: anything that is open collaborator content — your introduction, your project listing, your contribution log, your proposed new kinds-of-project. Anything you want changed in the canonical content (the kinds-of-project descriptions, the engagement mechanics, the conventions) goes through the meta-layer's standard maintainer-review process via a pull request to that repository.

## Meta-conventions

This project follows the [meta-conventions of the Resonance meta-layer](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront#meta-conventions), with one explicit override:

- **Atlas-linked maintainer-gating is OFF for this repo.** Atlas-linked pages here are openly editable. See "Open-edit policy" above.

All other meta-conventions apply unchanged:

- Append-only.
- Recursive atlas (this Atlas).
- Recursive document conventions.
- No duplication within an index.
- Every session and every subagent owns at least one master document.
- Anything external can be integrated by building a tool that brings it in.
- Archive is only for supersession (not for implementation).
