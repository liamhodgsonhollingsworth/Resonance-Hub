# Resonance Hub

The open collaborator entry point for the [Resonance meta-layer](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront) network. This repository is public and openly editable — anyone can fork, branch, and PR without prior permission. Collaborator listings, contributions, and proposed new kinds-of-project live natively here.

## Message to future collaborators

This system is not any one thing. It is a structure of knowledge, tools, and conventions that becomes whatever the person using it wants it to be. If you clone the meta-layer onto your own system, the same underlying machinery can run several distinct kinds of project — the ones listed under [Kinds of project](kinds_of_project/) plus whatever new shape you propose. To bootstrap a session against the meta-layer, paste this seed into a fresh Claude Code session running in a clone of the meta-layer repository:

    Load and orient to the Resonance github repository.

After the session loads, it reads the auto-loaded CLAUDE.md and the meta-conventions on the meta-layer README, then proceeds per the meta-layer's seed.md if longer orientation is needed.

## Open-edit policy

This repository overrides the meta-layer's atlas-linked maintainer-gating: anyone can edit pages here without prior maintainer permission. The repo does not use CODEOWNERS or branch-protection rules requiring maintainer review on PRs. The boundary: open content lives here; canonical content (kinds-of-project descriptions where they are linked from the meta-layer atlas, conventions, the README itself) is editable but proposals that change canonical content meaning should still surface through discussion before merging.
