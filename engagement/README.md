# How to engage

The mechanics of engaging with the Resonance project. Most of the canonical engagement mechanics live in the meta-layer's [hub of contribution](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/hubs/contribution.md#how-to-engage); this page links to each and provides space for collaborator-added engagement notes.

## What this index contains

Every channel through which a collaborator or contributor can engage with the project. The canonical descriptions live in the meta-layer; this page reflects them and adds collaborator-specific notes (tips for getting started, examples of contributions that worked well, etc.).

## Canonical engagement mechanics (link back to meta-layer)

- **[Feedback and comments](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/hubs/contribution.md#feedback-and-comments)** — github issues tagged with "feedback", PR review comments, issue comments referencing a page.
- **[Email suggestions](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/hubs/contribution.md#email-suggestions)** — direct email to the maintainer for longer notes or ideas not yet clear enough to file as issues.
- **[Contributing pages](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/hubs/contribution.md#contributing-pages)** — fork, branch, pull request; the standard github mechanics with project-specific area conventions.
- **[Permission model](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/hubs/contribution.md#permission-model)** — anyone can fork and PR; direct push is maintainer-only on most repos; this Resonance-Hub repo is the exception with open editing.
- **[Where contributions go](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/hubs/contribution.md#where-contributions-go)** — which area of which repo a contribution lands in, by kind.
