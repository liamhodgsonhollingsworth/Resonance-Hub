# Permission model

Anyone can fork any project repository and open a pull request against it. Direct push access is reserved for the maintainer and trusted long-term collaborators on most repos; the [Resonance Hub repository](https://github.com/liamhodgsonhollingsworth/Resonance-Hub) is the exception with open-edit policy.

The intended pattern is that contributors create new pages within specific areas of a repository rather than modify existing pages. This follows the append-only invariant: instead of changing what is here, add what you want to see. If a page needs to be updated, write a successor page and link to it from the original. Git history preserves the older version automatically.

For the formal edit-permission rule across repos see the [edit permissions convention](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/conventions/edit_permissions.md) in the meta-layer.

To edit this page, refer to [open-edit](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/conventions/editing/open_edit.md).
