# Where contributions go

The atlas-mirrored rule for what kind of contribution lands where:

- New project pages → a new project repo (linked from the meta-layer atlas as a top-level entry).
- New ideas → the meta-layer's `ideas/` directory. Feature ideas are a subset of ideas with more specific implementation detail.
- New conventions or refinements → the meta-layer's `conventions/` directory. Substantial changes to existing conventions should be made as new pages rather than edits.
- New archived entries → the meta-layer's `archive/` (only for supersession; implementation does not archive).
- New collaborator hubs or open-edit content → the [Resonance Hub repository](https://github.com/liamhodgsonhollingsworth/Resonance-Hub).
- New build state for a specific project → that project's `whats_built.md`.

Contributions that change foundational pages (the meta-layer README and meta-conventions, the licenses) require maintainer approval before the pull request is opened.

To edit this page, refer to [open-edit](https://github.com/liamhodgsonhollingsworth/The-Resonance-Wavefront/blob/main/conventions/editing/open_edit.md).
